# Broadcast Server

Broadcast Server - это простое приложение для чата на основе WebSocket, которое позволяет нескольким клиентам подключаться к серверу и обмениваться сообщениями в реальном времени.

## Описание

Broadcast Server реализует систему чата с использованием библиотеки `websockets` для Python. Приложение состоит из двух основных компонентов:

1. **Сервер**: Обрабатывает подключения клиентов, управляет списком активных подключений и транслирует сообщения всем подключенным клиентам.
2. **Клиент**: Консольное приложение для подключения к серверу и обмена сообщениями.

### Особенности

- Поддержка нескольких клиентов одновременно
- Уведомления о подключении/отключении пользователей
- Обмен текстовыми сообщениями в реальном времени
- Простой интерфейс командной строки
- JSON-формат для обмена данными между сервером и клиентами

## Установка

### Требования

- Python 3.13 или выше
- Poetry (для управления зависимостями)

### Установка зависимостей

```bash
# Клонируйте репозиторий
git clone https://github.com/kavlan-dev/broadcast-server.git
cd broadcast-server

# Установите зависимости с помощью Poetry
poetry install
```

## Настройка

Проект использует Poetry для управления зависимостями. Все необходимые настройки указаны в файле `pyproject.toml`.

### Основные параметры

- **Хост**: По умолчанию `localhost`
- **Порт**: По умолчанию `8765`
- **Имя пользователя**: По умолчанию `anonymous`

Вы можете изменить эти параметры при запуске сервера или клиента через аргументы командной строки.

## Запуск

### Запуск сервера

```bash
# Запуск сервера с параметрами по умолчанию
poetry run broadcast-server start

# Запуск сервера с пользовательскими параметрами
poetry run broadcast-server start --host 0.0.0.0 --port 8080
```

### Подключение клиента

```bash
# Подключение клиента с параметрами по умолчанию
poetry run broadcast-server connect

# Подключение клиента с пользовательскими параметрами
poetry run broadcast-server connect --host localhost --port 8765 --username MyUsername
```

## Использование

### Команды сервера

- `start`: Запускает сервер WebSocket
  - `--host`: Хост для привязки сервера (по умолчанию: localhost)
  - `--port`: Порт для привязки сервера (по умолчанию: 8765)

- `connect`: Подключается к серверу в качестве клиента
  - `--host`: Хост сервера (по умолчанию: localhost)
  - `--port`: Порт сервера (по умолчанию: 8765)
  - `--username`: Имя пользователя (по умолчанию: anonymous)

### Работа с клиентом

После подключения к серверу:

1. Введите сообщение и нажмите Enter для отправки
2. Все сообщения от других пользователей будут отображаться в реальном времени
3. Для выхода из клиента введите `выйти` или `exit`

### Пример использования

1. В одном терминале запустите сервер:
   ```bash
   poetry run broadcast-server start
   ```

2. В другом терминале подключите первого клиента:
   ```bash
   poetry run broadcast-server connect --username Alice
   ```

3. В третьем терминале подключите второго клиента:
   ```bash
   poetry run broadcast-server connect --username Bob
   ```

4. Теперь Alice и Bob могут обмениваться сообщениями через сервер.

### Основные компоненты

- **BroadcastServer**: Основной класс сервера, обрабатывающий WebSocket-соединения
- **ConnectionManager**: Управляет списком активных подключений и транслирует сообщения
- **CliClient**: Консольный клиент для взаимодействия с сервером
- **Connection**: Представляет одно подключение клиента
